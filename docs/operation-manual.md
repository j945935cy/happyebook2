# happyebook2 使用手冊

本手冊整理 happyebook2 目前可直接使用的操作方式，包含本機預覽、書籍維護、投稿審核、上線前檢查與已知限制。內容以目前專案實作為準，適合站台管理者、內容編輯與後續接手維護的工程師參考。

## 1. 專案定位

happyebook2 是一個靜態電子書平台，使用 HTML5、CSS3 與 Vanilla JavaScript 製作，主要提供：

- 網頁版電子書入口
- 免費與付費內容整理
- 作者投稿資訊收集
- 教學型與內容導向的書籍展示頁

目前前台書籍資料來源為 `src/books.json`，頁面會依這份資料顯示書籍卡片、標籤、分類、按鈕與單本作品頁。

## 2. 常用目錄與檔案

```text
happyebook2/
├─ assets/                    素材、封面圖、插圖
├─ docs/                      專案文件
│  └─ operation-manual.md     本手冊
├─ PROMPTS/                   開發提示詞
├─ src/                       前台頁面、樣式、腳本與資料
│  ├─ books.json              書籍資料來源
│  ├─ styles.css              全站樣式
│  ├─ script.js               前台互動
│  ├─ admin.html              管理頁
│  ├─ admin.js                管理頁互動
│  ├─ submit.html             投稿頁
│  └─ book.html               單本作品頁
├─ AGENTS.md                  專案規範
├─ index.html                 根目錄入口，會轉址到 /src/index.html
└─ readme.md                  專案總覽
```

## 3. 本機預覽方式

### 3.1 建議做法

由於前台會用 `fetch` 讀取 `books.json`，建議使用本機伺服器預覽，不要只用檔案總管直接開啟 HTML。

在專案根目錄執行：

```powershell
python -m http.server 8000
```

然後開啟：

```text
http://localhost:8000/src/
```

### 3.2 預覽時要檢查的頁面

- 首頁：`/src/index.html`
- 書籍列表：`/src/books.html`
- 單本作品頁：`/src/book.html?id=作品ID`
- 投稿頁：`/src/submit.html`
- 管理頁：`/src/admin.html`

### 3.3 預覽檢查重點

- 首頁精選作品是否正常顯示。
- 書籍列表的類型與分類篩選是否可用。
- 單本作品頁是否能用正確 `id` 載入。
- 手機版導覽列是否可正常展開與收合。
- 書封圖、按鈕連結與外部網址是否正常。

## 4. 書籍資料維護

### 4.1 資料檔案

所有正式上架書籍都維護在：

```text
src/books.json
```

每本書是一個 JSON 物件，常用欄位如下：

| 欄位 | 必填 | 說明 |
| --- | --- | --- |
| `id` | 是 | 唯一代號，建議用英文小寫、數字、連字號。 |
| `title` | 是 | 書名。 |
| `subtitle` | 否 | 副標題。 |
| `author` | 是 | 作者或單位名稱。 |
| `category` | 是 | 分類名稱，例如「英語學習」、「證照學習」。 |
| `type` | 是 | 類型，可用 `web`、`free`、`paid`。 |
| `format` | 是 | 顯示格式，例如「網站閱讀」、「PDF」、「EPUB」。 |
| `cover` | 是 | 封面圖路徑，通常為 `../assets/images/檔名.svg`。 |
| `description` | 是 | 作品簡介。 |
| `downloadUrl` | 視情況 | 免費下載連結。 |
| `buyUrl` | 視情況 | 購買連結。 |
| `readUrl` | 視情況 | 網頁閱讀或外部閱讀頁連結。 |
| `featured` | 否 | 是否列入首頁精選。 |
| `popular` | 否 | 是否列入首頁熱門。 |
| `priceLabel` | 建議填寫 | 前台顯示的取得方式文字，例如「免費閱讀」、「提供試閱版」。 |
| `published` | 否 | 是否公開顯示；未填時前台視為公開。 |

### 4.2 `type` 與按鈕對應

前台按鈕會依 `type` 決定主要連結：

- `web`：主要按鈕連到 `readUrl`
- `free`：主要按鈕連到 `downloadUrl`
- `paid`：主要按鈕連到 `buyUrl`

如果 `type` 設錯，前台按鈕就可能導到錯誤位置。

### 4.3 新增或修改作品

1. 開啟 `src/books.json`
2. 複製一筆既有資料作為範本
3. 修改 `id`、書名、分類、封面圖與連結
4. 確認 JSON 語法正確
5. 重新整理前台檢查顯示結果

維護時請注意：

- `id` 不可重複。
- `category` 名稱要統一，不要混用相近詞。
- 外部連結要填完整網址。
- 不想公開的作品請設定 `"published": false`。
- 若要出現在首頁，需額外設定 `featured` 或 `popular`。

## 5. 管理頁使用方式

### 5.1 管理頁用途

`src/admin.html` 是靜態輔助工具，方便做以下工作：

- 新增或編輯書籍資料
- 調整作品順序
- 切換公開或隱藏
- 匯出整理後的 `books.json`
- 查看 Google 表單回應後手動整理書籍資料

### 5.2 操作流程

1. 用本機伺服器開啟 `src/admin.html`
2. 在表單區新增或修改作品資料
3. 於清單區檢查排序、狀態與內容
4. 點選複製或下載 JSON
5. 手動把結果更新回 `src/books.json`
6. 回到前台頁面重新整理確認結果

### 5.3 重要限制

- 管理頁不會直接寫回專案檔案。
- 下載或複製 JSON 後，仍需人工更新 `src/books.json`。
- 管理頁不會自動讀寫 `src/books.json`，仍需人工整理與覆蓋檔案。

## 6. 投稿頁與待審資料

### 6.1 投稿頁目前的實際行為

`src/submit.html` 目前改為導向 Google 表單收件，不再把投稿資料暫存在瀏覽器。

這表示：

- 投稿資料會集中在 Google 表單與其回應表。
- 平台管理者可跨裝置查看收件資料。
- 正式上線前，需先在 `src/script.js` 設定 Google 表單與回應表網址。

### 6.2 投稿審核流程

1. 使用者在 `src/submit.html` 填表送出
2. Google 表單收集投稿資料並記錄在回應表
3. 版主在 `src/admin.html` 開啟回應表連結檢查投稿內容
4. 通過後手動整理到書籍編輯表單，再輸出 JSON
5. 將輸出的內容覆蓋 `src/books.json`
6. 重新整理前台頁面確認上架結果

### 6.3 正式上線建議

目前版本已適合先用 Google 表單正式收件；若後續需要更完整的權限、審核或自動化功能，可再評估以下方式：

- 串接 Google 表單
- 使用第三方表單服務
- 串接自有後端 API 或資料庫

## 7. 分類、首頁與單本頁的維護重點

### 7.1 首頁

首頁會根據 `featured` 顯示精選作品，並統計公開作品數、分類數與網頁版數量。

若作品未出現在首頁，請檢查：

- `published` 是否為 `false`
- `featured` 是否為 `true`
- `books.json` 是否格式錯誤

### 7.2 書籍列表頁

書籍列表頁會依 `type` 與 `category` 篩選，但目前 `src/books.html` 內的分類按鈕是手動寫死的。

這表示新增分類時，除了修改 `src/books.json`，還要同步檢查 `src/books.html` 是否需要補上新的分類按鈕。否則資料存在，但使用者可能無法直接用按鈕篩選到新分類。

### 7.3 單本作品頁

單本頁使用網址參數載入資料：

```text
book.html?id=grammar-beginners-v2
```

若網址中的 `id` 找不到對應作品，頁面目前會退回顯示第一本公開作品。上線前請務必抽查每張卡片的「更多資訊」連結。

## 8. 封面圖與素材管理

建議把封面圖集中放在：

```text
assets/images/
```

維護原則：

- 優先使用清楚易懂的檔名
- 儘量維持封面比例一致
- 從 `src/` 頁面引用時，路徑通常寫成 `../assets/images/檔名.svg`
- 若使用外部圖片網址，要確認來源穩定且可公開存取

新增封面後請檢查：

- 卡片有無破圖
- 單本頁封面是否正常
- 手機版是否裁切過多

## 9. 樣式與前端腳本維護原則

### 9.1 樣式

全站樣式集中於：

```text
src/styles.css
```

請遵守以下原則：

- 盡量沿用既有元件風格
- 使用 CSS 變數管理色彩
- 避免把大量樣式寫進 HTML
- 導覽列、按鈕、標籤、卡片、表單與 Footer 要維持一致

### 9.2 JavaScript

前台互動集中於：

```text
src/script.js
```

管理頁互動集中於：

```text
src/admin.js
```

請遵守以下原則：

- 不引入 React、Vue、Angular、jQuery 或其他大型框架
- 不把大量 inline JavaScript 寫進 HTML
- 修改篩選、資料載入或導覽互動時，要一起檢查首頁、列表頁與單本頁

## 10. 上線前檢查清單

### 10.1 內容檢查

- `src/books.json` 為合法 JSON
- 每本書的 `id` 唯一
- 書名、作者、分類與說明無錯字
- `type` 與連結欄位相符
- 不公開作品已設定 `"published": false`

### 10.2 畫面檢查

- 首頁、書籍列表、單本頁、投稿頁、關於頁、聯絡頁都能開啟
- 桌機、平板、手機沒有爆版
- 手機選單可以展開與收合
- 書籍卡片按鈕可點擊
- 所有封面圖正常顯示

### 10.3 結構檢查

- 大量 CSS 仍集中在 `src/styles.css`
- 大量互動仍集中在 `src/script.js` 與 `src/admin.js`
- 根目錄 `index.html` 可正確導向 `/src/index.html`
- 沒有引入被禁止的前端框架

## 11. 常見問題

### 11.1 書籍沒有出現

請檢查：

- `books.json` 是否有語法錯誤
- 是否使用本機伺服器開啟
- `published` 是否為 `false`
- 瀏覽器主控台是否有 `fetch` 錯誤

### 11.2 新分類無法用按鈕篩選

請檢查：

- `src/books.json` 的 `category` 名稱是否一致
- `src/books.html` 是否已新增對應的分類按鈕

### 11.3 投稿資料在管理頁看不到

請檢查：

- 是否在同一台裝置、同一個瀏覽器操作
- 是否清除了瀏覽器資料
- 是否曾使用無痕模式

### 11.4 管理頁出現亂碼

目前專案中的 `src/admin.html` 與 `src/admin.js` 確實存在部分文字亂碼，這不是手冊顯示問題，而是檔案內容本身需要後續整理。若要正式交付管理功能，建議另外排程修正。

## 12. 例行維護建議

每次調整作品後，建議依序執行：

1. 檢查 `src/books.json` 語法
2. 檢查首頁精選區
3. 檢查書籍列表篩選
4. 檢查單本作品頁與按鈕連結
5. 檢查手機版導覽列與卡片排版
6. 上線後用正式網址再抽查一次主要頁面

每月建議整理一次：

- 移除失效連結
- 檢查分類是否需要合併
- 更新投稿說明與聯絡資訊
- 補齊缺少的封面與摘要
- 檢查本手冊內容是否仍符合實際流程
